# Онлайн-кинотеатр — Модуль 1: Проектирование базы данных

## Описание системы

Информационная система «Онлайн-кинотеатр» предназначена для управления каталогом фильмов и подписками пользователей. Система реализует полный поток: регистрация → авторизация → оформление подписки → управление подписками через панель администратора.

## Стек технологий

| Компонент | Технология |
|---|---|
| Бэкенд | Go 1.26, Gin, sqlx |
| База данных | PostgreSQL 18 (Docker) |
| Миграции | golang-migrate |
| Авторизация | JWT (golang-jwt/jwt v5) |
| Хэширование паролей | bcrypt |
| Фронтенд | Next.js 16, TypeScript, Tailwind CSS |

## Сущности базы данных

### roles — роли пользователей
| Поле | Тип | Описание |
|---|---|---|
| id | SERIAL PK | Идентификатор |
| code | VARCHAR(50) UNIQUE | Код роли: admin, user |
| name | VARCHAR(100) | Название роли |

### users — пользователи
| Поле | Тип | Описание |
|---|---|---|
| id | BIGSERIAL PK | Идентификатор |
| login | VARCHAR(64) UNIQUE | Логин (латиница+цифры, ≥6 символов) |
| password_hash | TEXT | Хэш пароля (bcrypt) |
| full_name | VARCHAR(200) | ФИО |
| phone | VARCHAR(20) | Телефон в формате 8(XXX)XXX-XX-XX |
| email | VARCHAR(255) | Email |
| role_id | INT FK → roles.id | Роль пользователя |
| created_at | TIMESTAMP | Дата регистрации |

### movies — каталог фильмов
| Поле | Тип | Описание |
|---|---|---|
| id | SERIAL PK | Идентификатор |
| title | VARCHAR(200) | Название |
| description | TEXT | Описание |
| genre | VARCHAR(100) | Жанр |
| duration_min | INT | Длительность в минутах |
| age_rating | VARCHAR(10) | Возрастной рейтинг |
| poster_url | VARCHAR(500) | URL постера |

### subscriptions — подписки пользователей
| Поле | Тип | Описание |
|---|---|---|
| id | BIGSERIAL PK | Идентификатор |
| user_id | BIGINT FK → users.id | Пользователь |
| plan | VARCHAR(50) | Тариф: basic, standard, premium |
| payment_method | VARCHAR(20) | Способ оплаты: card, sbp |
| status | VARCHAR(20) | Статус подписки |
| started_at | TIMESTAMP | Дата начала |
| expires_at | TIMESTAMP | Дата окончания |

## Связи между сущностями

```
roles 1 ──── N users
users 1 ──── N subscriptions
```

## Статусы подписки

| Статус | Описание |
|---|---|
| new | Новая — подписка оформлена, ожидает активации |
| active | Активна — подписка действует |
| expired | Истекла — срок действия закончился |
| cancelled | Отменена — подписка отменена администратором |

## API эндпоинты

### Авторизация
| Метод | Путь | Описание |
|---|---|---|
| POST | /api/auth/register | Регистрация нового пользователя |
| POST | /api/auth/login | Вход, возвращает JWT токен |

### Пользователь (требует JWT)
| Метод | Путь | Описание |
|---|---|---|
| GET | /api/user/subscriptions | Список своих подписок |
| POST | /api/user/subscriptions | Оформить новую подписку |
| PATCH | /api/user/subscriptions/upgrade | Сменить тариф активной подписки |

### Администратор (требует JWT + роль admin)
| Метод | Путь | Описание |
|---|---|---|
| GET | /api/admin/subscriptions | Список всех подписок |
| PATCH | /api/admin/subscriptions/:id/status | Изменить статус подписки |

## Валидация при регистрации

- **Логин** — только латиница и цифры, минимум 6 символов, уникальность проверяется в БД
- **Пароль** — минимум 8 символов, требуется подтверждение
- **ФИО** — обязательное поле
- **Телефон** — формат 8(XXX)XXX-XX-XX, автоматическое форматирование при вводе
- **Email** — проверка формата регулярным выражением

## Инструкция по запуску

### Требования
- Go 1.26+
- Node.js 20+
- Docker

### Запуск базы данных
```bash
cd ~/dev/services/postgres
docker compose up -d
```

### Запуск бэкенда
```bash
cd backend
go run cmd/api/main.go
```

Бэкенд запускается на `http://localhost:8080`. При первом запуске автоматически применяются миграции и сиды.

### Запуск фронтенда
```bash
cd frontend
npm run dev
```

Фронтенд запускается на `http://localhost:3000`.

### Тестовые учётные записи

| Роль | Логин | Пароль |
|---|---|---|
| Администратор | admin | admin123 |
| Пользователь | user1 | admin123 |

## Сценарии использования

1. **Регистрация** — `/register` → заполнить форму → автоматический переход на `/login`
2. **Вход пользователя** — `/login` → переход на `/dashboard`
3. **Оформление подписки** — выбрать тариф, способ оплаты, период → нажать «Оформить»
4. **Апгрейд тарифа** — при наличии активной подписки форма меняется на форму смены тарифа
5. **Вход администратора** — `/login` → переход на `/admin`
6. **Управление подписками** — администратор видит все подписки и может менять статус через дропдаун
